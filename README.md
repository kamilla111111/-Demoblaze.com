## Выводы

Основная проблема, которую я увидела- это отсутствие серверной валидации. API принимает пустые поля, спецсимволы и потенциально опасные строки. Это делает систему уязвимой для атак и может привести к падению сервера.

Если бы такие проверки выполнялись на этапе разработки, большинства этих багов можно было бы избежать. Тестирование должно начинаться рано- это снижает риски и экономит время на исправления.
